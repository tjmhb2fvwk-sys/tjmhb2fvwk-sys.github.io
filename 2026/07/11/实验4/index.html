实验报告4：Linux基本操作

实验报告：Linux基本操作

1、实验目的

使用命令行进行文件导航、软件包安装等，了解和实践命令行的基本使用。

2、实验环境

  • 操作系统：Ubuntu Server 22.04 LTS（VirtualBox 虚拟机）
  • 虚拟机配置：node1（192.168.56.102），内存 2GB，硬盘 20GB
  • 前置条件：已完成实验三，虚拟机网络配置正常

3、实验过程

步骤一：登录与查看系统信息

登录 Ubuntu Server 后，依次执行 whoami 查看当前用户、uname -a 查看内核与架构信息、lsb_release -a 查看 Ubuntu 版本、hostname 查看主机名、pwd 查看当前工作目录。

步骤二：文件与目录的基本操作

执行 ls -lh 列出当前目录文件的详细信息，使用 cd ~ 切换到用户主目录。使用 mkdir test_dir 创建新目录，touch test.txt 创建空文件，cp test.txt test_copy.txt 复制文件，mv test.txt /tmp/ 移动文件，rm -f test.txt 删除文件。

⚠️ rm -rf 命令会永久删除文件且不可恢复，请谨慎使用！

步骤三：查看系统资源状态

使用 free -h 查看系统内存使用情况，df -h 查看磁盘使用情况，top 查看进程和资源占用（按 q 退出），du -sh * 查看当前目录下各子目录的大小。

步骤四：软件包管理

执行 sudo apt update 更新软件包列表，sudo apt install nginx -y 安装 Nginx 软件包，sudo apt remove nginx -y 删除已安装的软件包，sudo apt upgrade -y 升级所有已安装的软件包。

步骤五：配置主机名与 hosts 解析

使用 sudo hostnamectl set-hostname template 修改主机名为 template，编辑 /etc/hosts 文件添加各节点的 IP 与主机名映射关系。


图1：/etc/hosts 主机名解析配置

步骤六：验证多机环境

在 node1 上执行 ping -c 3 8.8.8.8 测试外网连通性，ping -c 3 192.168.56.1 测试宿主机连通性，ping -c 3 192.168.56.102 测试 node2 连通性，ping -c 3 node3 测试主机名解析。


图2：多机环境网络连通性测试

4、实验结果

测试项 目标 结果
外网连通 8.8.8.8 ✅ 成功
宿主机连通 192.168.56.1 ✅ 成功
虚拟机互连 192.168.56.102 ✅ 成功
主机名解析 node3 ✅ 成功

5、知识总结

命令 作用 示例
pwd 显示当前目录 pwd → /home/tang
ls -lh 列出文件详情 ls -lh /home
mkdir 创建目录 mkdir test_dir
touch 创建空文件 touch test.txt
sudo apt install 安装软件包 sudo apt install nginx -y

6、出现问题

问题1：执行 sudo apt update 时提示 “Temporary failure resolving”。

  • 原因：DNS 配置错误或网络不通。
  • 解决：检查 /etc/netplan/50-cloud-init.yaml 中 nameservers 配置，添加 8.8.8.8 和 114.114.114.114。

问题2：ping node3 显示 “Destination Host Unreachable”。

  • 原因：node3 未开机或 IP 地址配置错误。
  • 解决：启动 node3 虚拟机，确认 IP 地址为 192.168.56.104。

7、心得体会

通过本次实验，我掌握了 Linux 命令行的基本操作，包括文件导航、目录管理、文件操作和系统资源查看。apt 软件包管理让我体会到了 Linux 生态的便捷性。在配置 hosts 文件和验证多机网络的过程中，我加深了对网络配置的理解。命令行操作虽然初学时有些陌生，但其高效性让我体会到了 Linux 系统管理的魅力。【课程大纲-思政融合点1】
